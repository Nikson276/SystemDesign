# Сбор и анализ требований

## Глоссарий и сокращения

| Сокращение / Термин | Расшифровка / Описание |
|---------------------|------------------------|
| ТУНЕЦ               | Название музыкальной платформы |
| DAU                 | Daily Active Users — количество уникальных пользователей в день |
| MAU                 | Monthly Active Users — количество уникальных пользователей в месяц |
| UGC                 | User-Generated Content — контент, загружаемый пользователями |
| CDC / CDP           | Change Data Capture / Change Data Processing — это паттерн, который позволяет отслеживать изменения данных в источнике (insert/update/delete в Postgres) и передавать эти изменения в другие системы в реальном времени или почти в реальном времени |
| SLA                 | Service Level Agreement — соглашение об уровне обслуживания |
| DRM                 | Digital Rights Management — система управления цифровыми правами |
| CDN                 | Content Delivery Network — сеть доставки контента |
| NFR                 | Non-Functional Requirement — нефункциональное требование |
| API                 | Application Programming Interface — интерфейс прикладного программирования |
| SSO                 | Single Sign-On — единая точка входа |
| СБП                 | Система быстрых платежей (российская платёжная система) |
| USDT                | Криптовалюта Tether (стейблкоин, привязанный к USD) |

> **Допущение**:  
> На этапе проектирования предполагается, что весь музыкальный контент загружается **авторами через верифицированную админку**. Система проверки прав и лицензирования будет реализована на последующих этапах развития продукта.

## 1. Бизнес-цель и критерии успеха

- **Бизнес-цель**: монетизация через премиум-подписки и рекламу.  
- **Критерии успеха**: конверсия в премиум и доход от рекламы.  
- **География**: запуск в России и СНГ с поддержкой нескольких регионов.

## 2. Функциональные требования

### Основной функционал
- Офлайн-прослушивание: **не поддерживается**.
- Источник контента: **только авторы через админку** (UGC с верификацией).
- Поддержка подкастов/аудиокниг: **планируется в будущем**.
- Типы плейлистов: **публичные, приватные, совместные, системные**.
- Кроссплатформенное воспроизведение: **не поддерживается**.

### Реклама и монетизация
- Реклама показывается **каждые 7 минут** у free-пользователей.
- Free-пользователи имеют **полный доступ ко всем функциям**, кроме отключения рекламы.
- Типы подписок: **только базовая премиум-подписка** (без семейных/студенческих тарифов).

### Поиск и рекомендации
- Поиск поддерживает **фильтрацию по жанру, году, исполнителю и т.д.**  
- Рекомендации формируются **раз в сутки (батч-обработка)**.  
- Социальные функции: **только совместные плейлисты**.

## 3. Нефункциональные требования (НФ)

### Производительность и масштабируемость
- Целевой трафик: **1 000 000 DAU**, ~3 сессии на пользователя в день.
- Целевые latency:
  - поиск трека: **≤ 200 мс**,
  - загрузка трека: **≤ 1 сек**,
  - рекомендации: **≤ 200 мс**.
- Поддержка **>10 000 одновременных аудиопотоков** в регионе.

### Доступность и надёжность
- SLA: **99.8%** для API и медиапотока.
- Потеря данных (например, истории прослушиваний): **допустима**.

### Безопасность и комплаенс
- Соответствие ФЗ-152 / GDPR: **не требуется на текущем этапе**.
- DRM: **не используется**.
- Аутентификация: **через соцсети и email + пароль**.

### Ремонтопригодность и эксплуатация
- Облачная платформа: **Яндекс.Облако**.
- Стек: **Frontend — JavaScript; Backend — Python / Go**.
- Стратегия деплоя: **поддержка canary-релизов**.

## 4. Системные границы и интеграции

- **Интеграции**:
  - Импорт избранных треков из **Яндекс.Музыки и VK** (парсинг по согласию пользователя).
  - Платежи: **СБП и USDT**.
  - Аналитика: **внутренняя система**.
- **Ingestion pipeline**: контент загружается **авторами и админами через админку**, с последующей обработкой (транскодирование, метаданные).

## 5. Будущий роадмап продукта

- Платформы: **Telegram Mini App, веб, Android**.
- Аудиоформаты: **поддержка lossless (high-fidelity)** в будущем.
- AI-генерация контента: **не планируется**.

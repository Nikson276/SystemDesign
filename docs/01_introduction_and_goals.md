# Сбор и анализ требований

## Глоссарий и сокращения

| Сокращение / Термин | Расшифровка / Описание |
|---------------------|------------------------|
| ТУНЕЦ               | Название музыкальной платформы |
| DAU                 | Daily Active Users — количество уникальных пользователей в день |
| MAU                 | Monthly Active Users — количество уникальных пользователей в месяц |
| UGC                 | User-Generated Content — контент, загружаемый пользователями |
| SLA                 | Service Level Agreement — соглашение об уровне обслуживания |
| DRM                 | Digital Rights Management — система управления цифровыми правами |
| CDN                 | Content Delivery Network — сеть доставки контента |
| NFR                 | Non-Functional Requirement — нефункциональное требование |
| API                 | Application Programming Interface — интерфейс прикладного программирования |
| SSO                 | Single Sign-On — единая точка входа |

## 1. Бизнес-цель и критерии успеха

- Какова основная бизнес-цель платформы ТУНЕЦ? (монетизация, охват аудитории, конкуренция с существующими сервисами и т.д.)
- Какие метрики будут использоваться для оценки успеха? (DAU/MAU, время прослушивания, конверсия в премиум и т.п.)
- Есть ли ограничения по географии запуска (локальный рынок / глобальный)? Планируется ли поддержка нескольких регионов?

## 2. Функциональные требования

### Основной функционал
- Поддерживается ли офлайн-прослушивание для премиум-пользователей?
- Можно ли загружать треки пользователями (UGC), или платформа работает только с лицензированным контентом?
- Планируется ли поддержка подкастов или аудиокниг в будущем?
- Какие типы плейлистов поддерживаются? (публичные, приватные, совместные, генерируемые системой?)
- Возможна ли передача управления воспроизведением между устройствами (например, с телефона на умную колонку)?

### Реклама и монетизация
- Как часто показывается реклама? (между каждым треком / раз в N треков / по времени прослушивания?)
- Блокируется ли весь функционал (кроме прослушивания) для free-пользователей или только часть?
- Планируется ли гибкая система подписок (семейная, студенческая и т.д.)?

### Поиск и рекомендации
- Должен ли поиск поддерживать фильтрацию (по жанру, году, артисту и т.д.)?
- Насколько «живыми» должны быть рекомендации? (реального времени на основе текущего сеанса vs суточные батчи?)
- Планируется ли интеграция с социальными функциями (подписка на друзей, совместные плейлисты)?

## 3. Нефункциональные требования (НФ)

### Производительность и масштабируемость
- Ожидаемое количество пользователей на старте и через 12 месяцев?
- Какие целевые значения latency для:
  - поиска трека?
  - загрузки трека для воспроизведения?
  - формирования рекомендаций?
- Требуется ли поддержка >10 000 одновременных потоков в одном регионе?

### Доступность и надёжность
- Какой SLA ожидается для основного API и медиапотока? (например, 99.95% uptime)
- Допустима ли потеря данных (например, истории прослушиваний) при сбое?

### Безопасность и комплаенс
- Требуется ли поддержка локальных норм (ФЗ-152, GDPR для международных пользователей и т.д.)?
- Как обрабатываются права на контент? Используется ли DRM?
- Планируется ли аутентификация через соцсети, SSO или email + пароль?

### Ремонтопригодность и эксплуатация
- Предпочтительная облачная платформа? (Яндекс.Облако / on-prem / другое?)
- Есть ли предпочтения по языкам реализации или фреймворкам?
- Нужна ли поддержка blue/green или canary-развёртываний?

## 4. Системные границы и интеграции

- С какими внешними системами должна интегрироваться платформа?
  - Платёжные системы (Яндекс.Платёж, СБП, другие локальные провайдеры)?
  - Музыкальные лицензиары / дистрибьюторы?
  - Системы аналитики (внутренние, Яндекс.Метрика и т.д.)?
- Кто отвечает за загрузку и обработку музыкального контента? Есть ли отдельный ingestion pipeline?

## 5. Будущий роадмап продукта

- Планируются ли дополнительные платформы (Smart TV, автомобильные системы, умные колонки)?
- Ожидается ли поддержка high-fidelity аудио (lossless, spatial audio)?
- Есть ли планы по внедрению AI-генерируемого контента или AI-диджеев?
